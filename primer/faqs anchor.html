What is v17e?

v17e is the mobile optimized version of the v17 design system for ibm.com. It's a framework that will enable you to create a cohesive and engaging digital experience on the ibm.com domain. It's a foundation you can build on top of with extreme flexibility and efficiency. It's a set of tools and resources to help you focus on what really matters, the customer.

How does v17e differ from v17?
Adaptive grids for desktop, tablet, and mobile screens. Other enhancements: updated jquery code, icons have been converted to fonts, IBM custom fonts, as well as new, adaptive modules.

What's new in v17e? (details)
There are a few new classes and containers in v17e that you can use to improve your page appearance for mobile devices or just generally to tweak the page. Three biggies include 1) adding the ibm-resize class to an image tag allows the image to get smaller (or larger) as needed based on window size, 2) the ibm-band class puts an edge to edge band around ibm-columns, visually breaking the space into horizontal bands and opening up the space to the left and right of the content for use, and 3) ibm-sprite lets you have an image that looks different on mouse over or down using a single image (sprite).

Will non-mobile users benefit?
Yes, experience on desktop for retina screens has been improved. If a non-mobile user changes the size of the browser window the content will adjust/respond to that change. Additionally pages should load more quickly due to a number of changes.

Does v17e improve performance?
Improved performance is being delivered through a number of techniques including 1) splitting up the CSS file and only delivering the ones that are needed for the functions that are actually used on each page, 2) eliminating duplicate files on different ibm.com sites and combining different JS files so that there are fewer calls and better use of caching, 3) ensuring every file is compressed, 4) using CSS pre-processor to optimize CSS code, 5) changing graphical icons to fonts, 6) removing code and CSS that are no longer needed (including for V16) and 7) ensuring all centrally delivered JS and CSS files are delivered via Akamai's geographically distributed CDN Network of servers, to name a few.

Why weren't these improvements added to v17?
Many things have been rewritten from the ground up. Mandatory changes would have to be made to 100% of all ibm.com pages, and with the number of pages on v17 today, it would be impractical to coordinate and test all at once. For this reason it was determined that making this one mid-course correction (and adding in a large number of additional enhancements at the same time) is a more reasonable solution.

Is v17e a replacement for the 'Lite Blue' mobile proxy or can the two be used together?
Yes, v17e is replacing Lite Blue and no it is not compatible with the mobile proxy. If your site is currently using the mobile proxy, you will need to check and remove any go-mobile-type .js references. Use <link rel="IBM.Mobile.redirect" href="http://www.ibm.com/[UPDATE]/" /> to override 'Lite Blue', causing it to redirect users to an "alternate mobile version of a page." The "[UPDATE]" portion of that URL should have the canonical URI for the page. That way, if the user accidentally lands at m.ibm.com, they will be redirected back to www.ibm.com instead. To do all this you can learn more about [the Lite Blue proxy and the IBM.Mobile.redirect] and/or reach out to the IBM Corporate Webmasters for help.

How much work is involved in changing a v17 page into a v17e page?
Depending on your pages, potentially not a lot of work. For example, these would be the minimal steps that would be needed to migrate a page to v17e. To take advantage of some advanced v17e functions will require some additional work.
  1. Add the viewport meta tag
    e.g.
    <meta name="viewport" content="width=device-width, initial-scale=1"/>
  2. Change the .css file you are calling.
  3. For non-DBCS pages, add the "ibm-type" class to the page's body tag (e.g. change the body tag from <body id="ibm-com"> to <body id="ibm-com" class="ibm-type">). The "ibm-type" class applies the custom IBM fonts. This must not be applied to DBCS pages.
  4. The default template (left nav template) now requires an "ibm-default" class
    e.g.
    <div id="ibm-top"> would become <div id="ibm-top" class="ibm-default">
  5. If the page includes any of the following: a data table, list modules, form, application, and/or social icons for the content space, you'll need to add a call to the appropriate css file.

Where are the main JS and CSS files?
Once you are granted access to SVN, you will then be able to download the appropriate JS and CSS files from the SVN directory. They are located at v17_enhanced/trunk/css and v17_enhanced/trunk/js. The production references are: <link href="//1.www.s81c.com/common/v17e/css/ww.css" rel="stylesheet" title="www" type="text/css" /> and <script src="//1.www.s81c.com/common/v17e/js/ww.js" type="text/javascript"></script>. jQuery and Modernizr are included as part of the common v17e JS.

How do I modify or add my own custom components to v17e?
Some sites, by their nature, may need to have custom components to accomplish their business needs. In v17e there is a way to add custom components. Custom components are added through a site specific CSS file (ww_cci.css). Inside of this file you can define the custom styles by using a '.ibm_cci-' prefix followed by the modified module (e.g. .ibm_cci-leadspace). All custom modules must be reviewed (by the v17e team) prior to start of coding and approved after coding is complete. Customizations that are approved can be added to your site specific ww_cci.css file.
