Overlays

Overview

Overlays should not be confused with secondary browsers or pop-up windows, though all three are supported under v17.

  • Overlays are modal panels that are activated from within an HTML page. Overlays are produced with a simple Javascript script. When the user clicks a link, the overlay is generated and displays information that is contained in that HTML page. Note: As of 25-January-2013 the <h1> tag for overlay titles has been replaced with the <h2> to improve SEO ratings.
  • Secondary browsers (also known as secondary windows) are second instances of an Internet browser window that, when the user clicks a link, opens and displays information from a separate HTML file.
  • Pop-up windows also known as Secondary windows are also second instances of an Internet browser window that, when the user clicks a link, opens and displays information from a separate HTML file. However, the second instance of the browser displays a modified masthead and footer.

Secondary browsers and pop-up windows can be confusing and frustrating to users when implemented without proper care. Whenever possible, designers should use an overlay instead.

Overlay limitations

  • Overlays cannot be printed, emailed or bookmarked.
  • Overlays do not stay open between pages.
  • Overlays limit users' ability to access content on the underlying page.
  • Browser security constraints make multi-step transactions in overlays difficult. For example, the page must reload an e-mail this page application to show another overlay with the response.

Requirements

Pages that do not adhere to the following requirements may be delayed or disallowed.

  • Do not open overlays or secondary windows automatically unless they contribute directly to task completion, such as with error messages or instructions for how to proceed to the next step in a process.
  • Do not use automatically opening overlays or secondary windows for advertisements without prior approval. To request approval, please contact the UED Standards team.

Approved overlays and code examples

The code below illustrates how to create a link that, when selected, launches a specific overlay. In the example code, replace the string "overlay-1" with the name of your overlay. Note there are two instances of the overlay name in the code sample.

<!-- OVERLAY LINK EXAMPLE -->
<a class="ibm-popup-link" href="#overlay-1" onclick="ibmweb.overlay.show('overlay-1',this);return false;">Click here to view overlay example</a>

Below are several examples of approved overlay designs and their associated HTML code. The code can be placed anywhere within the <body> tag of the page. For each overlay, the first line of code has a generic id value listed (e.g., id="overlay-1"). Replace the id value with the name of the overlay you want to call. You can have multiple overlays on one page as long as each overlay name is unique. Also, you may have links within overlays, but those links must not launch overlays.
